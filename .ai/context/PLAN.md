## Current Stage

- Scope Source: `SCOPE.md` 缺失，按 `PROJECT_BRIEF + CURRENT_STATUS + DECISIONS + 最新用户指令` 推导。

## Scope Validation

### In Scope (S3-Adjust)
- 保持业务域名不变：认证/拉库/播放 URL 继续使用用户自己的 Emby URL。
- 引入“CF 优选参考域名 -> 候选节点 IP”能力，仅用于为 Emby 域名请求提供更快的节点选择。
- 网络硬约束：仅支持 IPv4 优选；不引入 IPv6（v6）。
- 下载调度按用户口径落地：
  - 当前曲目“已下载可播时长 < 30s”继续下载；
  - 当前曲目“已下载可播时长 > 30s”暂停下载；
  - 当“当前曲目剩余播放时长 < 30s”时，先下完当前剩余，再预下载下一曲前 30s，然后重复上述规则。
- 回滚并废弃本轮错误实现（错误理解“优选域名直接播放”和“缓冲抖动切策略”）。

### Out of Scope (S3-Adjust)
- 将第三方优选域名直接作为播放/下载业务域名。
- IPv6 优选链路（车机不支持）。
- “边下边播”改成“一直不停下载到整首完成”的固定策略。
- Media3、minSdk 升级、多协议扩展与完整离线媒体库改造。

### Success Criteria
- 请求日志可证明：业务请求 Host 为用户 Emby 域名，且可观察到优选 IP 命中/回退。
- 下载调度行为满足 30s 规则，不再出现错误策略导致的异常停切。
- API17 设备上首播与切歌体感改善，且不引入新崩溃。

## Workstreams

### W0 基线回滚
- 目标: 清除错误实现，恢复可控基线。
- 产出: 本地代码恢复到讨论前状态，仅保留文档规划改动。

### W1 CF 优选 IPv4 接线
- 目标: 为 Emby 域名请求接入“参考域名驱动的候选 IPv4 优选”。
- 产出: 可复用的解析/优选/回退链路（优选失败自动回退系统 DNS）。

### W2 30s 下载窗口调度
- 目标: 将下载行为改为“按可播窗口与剩余时长驱动”的暂停/恢复调度。
- 产出: 当前曲目与下一曲预下载的状态机实现。

### W3 诊断日志与验证口径
- 目标: 提供可复盘日志（不含敏感信息），支撑实机验证。
- 产出: 优选 IP 命中、下载暂停/恢复、可播时长估算、下一曲预下载阶段日志。

### W4 API17 回归验证
- 目标: 在目标车机上完成专项验证并回填证据。
- 产出: PASS/FAIL 结论、复现步骤与后续修复入口。

## Dependencies
- `W0 -> W1 -> W2 -> W3 -> W4`
- `W2` 依赖 `W1` 提供稳定网络入口。
- `W4` 依赖 `W1~W3` 完整落地。

## Risks & Assumptions
- 风险: 第三方参考域名可用性波动会影响优选稳定性（需回退策略兜底）。
- 风险: “下载字节 -> 可播秒数”估算在极端码率波动时有偏差。
- 假设: Emby 音频元数据可获取到时长，且下载响应可获取内容长度或可估算区间。
- 假设: 车机网络环境对 IPv4 优选收益明显高于默认解析。
