- 先建设 AI 协作工作台，再做功能实现。

- 未确认内容显式标记为“待确认/假设”。

- A1. 初始化核心文档与执行规则。

- A3. 建立任务队列与状态回写机制。

- D3. 错误处理与重试策略。

## Phase E - 歌词链路 (Pending)
- E1. 嵌入歌词读取（标准同步标签）。
- E2. 缺词异步 LrcApi 拉取，不阻塞播放主线程。
- E3. 歌词缓存（TTL 按天）与多语言显示（中文优先）。

## Phase F - UI 落地与性能收敛 (Pending)
- F1. 左侧导航与核心页面模板。
- F2. 轻玻璃拟态和低动效。
- F3. 低内存/低算力设备性能优化与回归测试。

## Phase G - 打包与交付 (Pending)
- G1. GitHub Actions Android 打包流程（APK/AAB）配置。
- G2. 签名与密钥管理（GitHub Secrets）规范。
